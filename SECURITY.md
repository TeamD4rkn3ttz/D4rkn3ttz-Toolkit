# Security Policy

## Supported versions

This repository distributes text-based skills and supporting validation code. Security fixes are applied to the latest revision of the `main` branch.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability, exposed secret, unsafe workflow, or privacy-sensitive defect.

Use GitHub's private vulnerability reporting for this repository:

<https://github.com/TeamD4rkn3ttz/D4rkn3ttz-Toolkit/security/advisories/new>

Include the affected file or workflow, impact, reproduction details, and a safe proof of concept when possible. Do not include real credentials, personal data, stolen material, or active malicious payloads.

Maintainers will acknowledge a complete report when practicable, investigate it, and coordinate disclosure and remediation through the private advisory. Please allow a reasonable remediation period before public disclosure.

## Scope

Reports about repository workflows, validation scripts, unsafe skill instructions, dependency integrity, or accidental sensitive-data exposure are in scope. General threat intelligence requests and third-party platform vulnerabilities are not.

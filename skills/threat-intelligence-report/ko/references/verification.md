# 조사·검증 기준

## 출처 계층

1. 원 게시물, 원본 샘플, 공식 권고, 피해 조직 공지
2. 기술 분석 보고서, 법 집행·정부기관 자료, 연구 논문
3. 신뢰할 수 있는 언론의 독립 취재
4. 재게시 계정, 집계 사이트, 커뮤니티 논평

낮은 계층도 최초 발견과 맥락 파악에는 쓸 수 있지만, 강한 결론의 유일한 근거로 쓰지 않는다.

## 유출 주장 검증

- 형식: 스키마, 필드명, 인코딩, 날짜·전화·주소 형식이 대상 서비스와 일관적인가?
- 값: 실제 도메인, 지역, 상품, 계정 규칙과 맞는가? 테스트 값이나 규칙적 생성 흔적이 있는가?
- 규모: 주장한 행 수와 파일 크기·필드 수·샘플 분포가 양립하는가?
- 중복: 과거 유출, 공개 데이터, 마케팅 DB, 테스트 데이터와 겹치는가?
- 신선도: 최근 생성일, 주문일, 활동 기록 등 시간 단서가 존재하고 일관적인가?
- 원천: 최초 게시자는 누구이며 재게시·판매·홍보 관계는 무엇인가?
- 평판: 판매 이력, 관리자 인증, PGP, 에스크로, 구매자 반응이 있는가?
- 피해성: 계정 탈취, 피싱, 사칭, 금융 피해 등에 바로 악용 가능한가?

대표적인 적신호: `example.com`, 연속 번호, 존재하지 않는 주소, 필드만 그럴싸한 데이터, 모든 값이 동일 패턴, 과거 샘플과 완전 일치, 최초 출처 없는 반복 재게시. 적신호 하나만으로 가짜를 확정하지 않는다.

## 위협 캠페인·악성코드 검증

- 최초 보고와 후속 보고의 날짜를 분리한다.
- 침투 벡터, 실행 흐름, 지속성, C2, 페이로드를 출처별로 대조한다.
- IOC가 샘플 고유인지 공유 인프라인지 구분한다.
- ATT&CK 매핑은 관찰된 행위에만 적용하고 과도하게 확장하지 않는다.
- 벤더가 붙인 그룹명과 다른 벤더의 별칭을 동일 조직으로 자동 병합하지 않는다.
- CVE, 패치, 영향 제품·버전은 공식 권고나 제조사 문서로 확인한다.

## 다크웹·텔레그램 검증

- 텔레그램은 원천이 아니라 증폭·홍보 채널일 수 있음을 전제로 한다.
- 채널명·로고·Reported by 문구를 신뢰 근거로 보지 않는다.
- 삭제, 복제, 사칭 채널 가능성을 기록한다.
- 행위자의 역할을 침투자, 판매자, 브로커, 운영자, 홍보·재게시자로 구분한다.
- 가입·구매·DM·파일 실행 등 위험하거나 불법일 수 있는 행위를 요구하지 않는다.

## 판정 체계

| 판정 | 기준 |
|---|---|
| 확인 | 독립적인 강한 근거가 결론을 지지하고 중대한 반증이 없음 |
| 부분 확인 | 핵심 일부는 확인됐으나 규모·신선도·원천 등 일부가 미확인 |
| 과거 자료 재포장 | 기존 공개·유출 자료와 실질적으로 동일 |
| 합성/조작 정황 | 다수의 비현실적 패턴과 반증이 있으나 완전한 원본 확인은 제한됨 |
| 미확인 | 결론을 낼 증거가 부족하거나 상충함 |

신뢰도는 판정과 별도로 `높음/중간/낮음`을 표시한다.

## 위험도 평가

각 항목 0~5점으로 내부 비교에 사용한다.

- 증거 강도
- 데이터·자산 민감도
- 신선도
- 확산 범위
- 악용 가능성

총점은 자동 결론이 아니라 우선순위 대화를 위한 보조값이다. 점수와 함께 한 문장 근거를 남긴다.

## 증거 취급

- 원본과 가공본을 분리한다.
- 확인 일시, 수집 위치, 계정, 게시물 ID, 타임존을 기록한다.
- 파일은 가능한 경우 해시를 남긴다.
- 샘플은 필요한 최소 범위만 다루고 개인정보를 마스킹한다.
- 공개 결과물에는 재식별·재악용이 가능한 내용을 제거한다.
- 캡처만 있고 원 URL과 시각이 없으면 증거 한계를 명시한다.
